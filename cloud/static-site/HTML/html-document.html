HTML document

Definition

Internet is a connected networks with published content.

HTML document is a main text content unit in Internet.

HTML stands Hypertext Markup Language.

Hypertext defines links to external content.

Markup defines semantic (meaning) structure of document content.

HTML element

HTML has elements enclosing or wrap different parts of the content for their semantic meaning definition.

HTML element has opening tag, closing tag and content.

Nested element can be defined in content of parent element. Nested elements are fully enclosed in parent element content.

Empty element does not have content. Empty element has no closing tag or nested element. Empty elements can be used for context definition or embeding other documents.

HTML Element structure

  1. Opening tag consists element name wrapped in angle brackets with optional attributes.
    1. Element name defines type of element. It is not case-sensitive.
    2. Attribute has an extra information (context) that is not in content. It has spaces, name and value
      • Spaces separate attribute from element name and other attributes
      • Name identifies value in context of element. It is followed by separating equal sign
      • Value provides specific context to element. It is wrapped by quotation marks because value can have spaces.
  2. Closing tag has same name like in opening tag with forward slash and no attributes

HTML document structure

HTML document has predefined set of elements:
  1. <!DOCTYPE html> is preamble defying applied set of rules
  2. <html> is a root element that wraps entire document content. Using attribute "lang" is encouraged so browser tools (like aids speech, translation and so forth) can determine right settings. This element has two child elements
    1. <head> is a container for document metadata. It is elements describing document having no displayed content.
    2. <body> is a root container for document content.

Document metadata

  • <meta charset="utf-8"> is element defying charset for text content
  • <title> - element defying entire document label. It can be different from first heading that does not have to stand alone taken out of context.

Sections

Article

<article> is an independent item of content (like newspaper article, blog entry, user submitted comment or any other. Every document is article by itself, but element is used if document has more than one self-contained items. Articles can be re-arranged if needed without loosing content meaning. Each article can have own header and footer.

Section

<section> is thematical grouping of content having own heading. In compare with <article> it is not independent, and order of sections are more determined.

Headings

Section heading can be structured in 6 heading levels with elements from <h1> to <h6>. It is assumed that <h1> element enclosing main title for section.

Grouping content

Paragraphs Element <p> used for marking up paragraphs of text. Also, it can be used like more general thematic grouping.

Lists Each line in list marked by element <li>. Line items supposed to be grouped in one of two list type elements

Unordered list use parent element <ul>

Ordered list is grouped by parent element <ol>

Text-level semantics

Elements on intra-paragraph level mark ups. Historically, text-level elements used for content representation definition (like using italic font and etc.) With introducing CSS, these elements supposed to be used for semantical definitions only, since Style sheet can change default appearance of any element. The following are some examples.
  • <em> element for content stress emphasis
  • <strong> element for showing words' importance
  • <q> element for marking quotations.
  • <i> element represents text in alternate voice or mood. Default historical italic format can be changed with Style sheet.
  • <br> is a line break that is part of content, as in poems or addresses. It must not be used for separating thematic groups in paragraph. Style sheet can change default "moving text to new line" rendering on something else like "adding extra spacing" or inserting special symbols.

Links

Using links creates network (web) of documents. Links can be used to augment current document or be a hyperlink to another document that can be visited in browser.

Element <a> (anchor) defines hyperlink labeled by its contents. Content of this element cannot be another anchor. Content cannot be interactive, so the only interaction defined by anchor is related to new content navigation. Attribute "hypertext reference" (href) provides unified reference link (URL) of another content.

Embedded content

These elements can embed different type of media (image, video, audio or any special) files into content of HTML document.

For example, element <img> embeds images into page. Element position defines image place in text content. This element has two main attributes:

  • source (src) defines path to image file
  • alternative (alt) Provides text substitute for image. It can be used for visual impairment screen reader and displayed in case of problem with image file loading.
